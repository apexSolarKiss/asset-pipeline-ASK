# Airtable Collection / Merchandising Proof Plan v1

## Purpose Of The Proof

This document defines one bounded collection / merchandising Airtable proof as the next honest proof move after the SKU-driven Furniture v1 chain and the clean campaign / editorial Cycle-1-equivalent rerun.

Its purpose is to test whether the current Airtable proof shape can carry one bounded collection / merchandising burden without schema revision, Airtable configuration revision, or a broader mode rollout.

This is a Cycle-1-equivalent planning artifact for a third distinct burden shape. It is not an execution artifact and it is not a generalized third-mode model attempt.

## Why Collection / Merchandising Is The Next Honest Mode

Current `main` already supports three narrow planning-level conclusions:

- SKU-driven Furniture v1 has been pressure-tested through multiple bounded cycles
- campaign / editorial has now cleared one bounded Cycle-1-equivalent proof cleanly
- prompt-artifact continuity remains explicitly on hold rather than reopened

That means the next honest question is not whether to widen campaign work or revisit the Cycle 4 prompt-artifact pressure.

The next honest question is whether the current packet-and-seam proof shape can carry a third distinct burden that is neither:

- single-item SKU truth
- nor campaign-family coherence with load-bearing review-board context

Collection / merchandising is the strongest next candidate because current repo-local truth already frames it as a grouped-set burden where:

- adjacency and spacing matter as governed relationships
- item-level truth can pull against grouped coherence
- review context appears useful but not load-bearing in the stronger campaign/editorial sense

This makes collection / merchandising the next honest bounded proof question without widening toward generalized third-mode rollout.

## Bounded Scenario

The proof should use one narrow collection / merchandising scenario only:

- one calm living-room grouping
- one small set of four items
- one anchor item reusing the existing governed product `SKU-CHAIR-002`

The proposed set is:

- anchor item: `SKU-CHAIR-002`
- one small side table
- one floor lamp
- one low storage bench

The point of this set is not room storytelling. The point is to test whether the grouped set can read as one governed merchandising arrangement whose adjacency and spacing remain legible while each item still remains sufficiently itself.

This keeps the proof closer to collection logic than to campaign or editorial room-world logic.

## Grouped-Set Burden

The grouped-set burden for this proof is:

- adjacency must read intentional rather than accidental
- spacing must preserve grouped coherence without crowding or detaching items
- positional emphasis must not collapse the set back into one hero SKU with incidental supporting props
- each item must remain legible enough as itself that grouped coherence does not erase item-level truth

The key tension to test is:

- item-level truth pulling against grouped coherence

That is the distinct burden this proof should carry. The live question is whether the current Airtable proof shape can hold one relationship-bearing set whose members remain both governably grouped and individually truthful enough.

## Role Of Review Context

This proof should treat review context as useful, but not campaign-style load-bearing.

The planned review context is:

- one grouped-set review board or contact-sheet-style comparison artifact
- optional light annotation only if spacing or adjacency concerns need to be called out for review clarity

That review context should help grouped-set judgment, but it should not become the primary readiness burden in the stronger campaign/editorial sense.

This keeps the proof honest to current repo-local evidence, which supports useful review context here without yet supporting campaign-side review-board doctrine.

## How This Burden Differs Materially From SKU And Campaign

This proof should frame its burden concretely rather than abstractly.

Relative to SKU-driven product imagery:

- the governing burden is no longer one item's truth and repeatable comparability alone
- the governing burden is whether several items still read as one controlled merchandising grouping

Relative to campaign / editorial:

- the governing burden is not broader brand-world coherence or review-board-supported family judgment across expressive assets
- the governing burden is tighter and more relational: grouped-set spacing, adjacency, and positional balance must stay legible without drifting into campaign-like room story logic

The bounded scenario should therefore test:

- governed grouping, not single-item hero truth
- governed adjacency, not campaign-family continuity

## First Seam Trail To Test

The first honest seam trail for this proof should be:

2. `transformation_check`

This should remain a first-pass proof only.

The seam trail is meant to answer whether one bounded collection / merchandising packet can:

- enter the Airtable surface cleanly
- preserve grouped-set burden across transformation
- produce governed outputs whose adjacency and spacing remain legible enough to validate
- reach readiness with useful review context that supports grouped-set judgment without becoming structurally primary

## Pre-Registered Outcome Classes

This proof should pre-register the same three narrow outcome classes already used in the campaign planning / findings chain:

- `findings-only`
  - the current proof shape carries the bounded collection / merchandising burden far enough to generate interpretable first-pass findings without requiring schema/configuration change
- `spec-earned pressure`
  - the current proof shape reveals a localized structural burden specific enough to justify later options/decision discussion, but not implementation in this pass
- `insufficient evidence`
  - the run does not produce clean enough evidence to distinguish whether the burden belongs to mode structure, execution capture, review-context thinness, or some other confounder

- the repo still appears to share one packet-and-seam model across distinct burden shapes

The live proof question is therefore not whether collection / merchandising deserves schema pre-adjustment in theory.

The live proof question is whether one bounded grouped-set packet can be declared and carried honestly enough on the current surface to yield interpretable first-pass findings.
