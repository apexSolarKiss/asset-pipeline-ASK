# Four-Mode Truth And Constraint Pressure Note

## Purpose

This note updates the repo's comparison layer from partial two-mode and three-mode framing to the full four worked-example modes now present on `main`.

Its purpose is to state only what the current examples now support structurally without turning that support into schema branching, taxonomy settlement, validator design, or new seam logic.

## Why A Four-Mode Comparison Is Now Possible

The repo now has four worked-example tracks that place distinct governing pressure inside the same packet-and-seam structure:

- `SKU-driven product imagery`
- `collection / merchandising`
- `marketing / message-driven`
- `brand campaign / editorial`

That means the four-mode structure is now earned by the repo's examples rather than merely asserted as a working taxonomy label.

## Single-Item Truth

In the SKU-driven case, pressure concentrates on one item and its repeatable comparability.

The strongest governing burden is that one product remains legible, faithful, and stable enough to support controlled downstream use across repeated outputs.

## Relational Truth

In the collection / merchandising case, pressure concentrates on governed relationships across multiple items.

The strongest governing burden is not one hero item alone, but whether adjacency, spacing, pairing, and positional emphasis keep the grouped set legible as one coherent merchandising arrangement without losing item-level truth.

## Message-Bearing Discipline

In the marketing / message-driven case, pressure concentrates on targeted communication purpose and message-bearing signal persistence.

The strongest governing burden is whether the asset family still reads as one usable promotional communication set whose message survives clearly enough across the intended controlled surfaces.

## Campaign Coherence

In the campaign / editorial case, pressure concentrates on bounded coherence across broader expressive latitude.

The strongest governing burden is whether multiple assets remain recognizably part of one campaign family while preserving required signals and staying inside the approved directional frame.

## What Still Appears Shared Across All Four

Despite those differences, the current examples still appear to sit inside one shared structure:

- inputs and context establish what the work is about
- constraints define what later work must remain within
- governed outputs must be sufficient for controlled downstream use
- approval-facing readiness remains distinct from automated approval itself

The strongest current claim is not that the four modes use the same governing pressure, but that they place different pressures inside one shared packet-and-seam model.

## What This Comparison Still Does Not Justify

This four-mode comparison does not by itself justify:

- new seams
- schema branching
- generalized workflow taxonomy settlement
- validator or code changes

## What This Note Does Not Do

- define field catalogs
- formalize schema
- introduce runtime or orchestration language
- imply validator or code changes
- claim that the four modes are a final industry taxonomy
