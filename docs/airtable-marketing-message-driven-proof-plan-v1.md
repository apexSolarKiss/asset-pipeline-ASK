# Airtable Marketing / Message-Driven Proof Plan v1

## Purpose Of The Proof

This document defines one bounded marketing / message-driven Airtable proof as the next honest proof move after the SKU-driven Furniture v1 chain, the clean campaign / editorial Cycle-1-equivalent rerun, and the clean collection / merchandising Cycle-1-equivalent proof.

Its purpose is to test whether the current Airtable proof shape can carry one bounded marketing / message-driven burden without schema revision, Airtable configuration revision, or a broader mode rollout.

This is a Cycle-1-equivalent planning artifact for the remaining obvious worked-mode Airtable proof gap. It is not an execution artifact and it is not a generalized marketing-mode model attempt.

## Why Marketing / Message-Driven Is The Next Honest Mode

Current `main` already supports four narrow planning-level conclusions:

- SKU-driven Furniture v1 has been pressure-tested through multiple bounded cycles
- campaign / editorial has now cleared one bounded Cycle-1-equivalent proof cleanly
- collection / merchandising has now cleared one bounded Cycle-1-equivalent proof cleanly
- prompt-artifact continuity remains explicitly on hold rather than reopened

That means the next honest question is not whether to widen campaign work, widen collection work, or revisit the Cycle 4 prompt-artifact pressure.

The next honest question is whether the current packet-and-seam proof shape can carry the remaining distinct burden already present in the repo-local worked examples:

- a targeted promotional communication set whose governing pressure is message persistence rather than single-item fidelity, grouped-set relationship, or campaign-family coherence

Marketing / message-driven is therefore the next honest bounded proof question. It is the remaining worked-example mode that has packet-and-bridge artifacts on `main` but no corresponding bounded Airtable proof plan or findings chain yet.

## Bounded Scenario

The proof should use one narrow marketing / message-driven scenario only:

- one limited-time seating promotion
- one controlled message unit: `Quiet Weekend Seating Event`
- one small three-asset promotional set
- one anchor product reusing the existing governed product `SKU-CHAIR-002`

The proposed promotional set is:

- anchor product: `SKU-CHAIR-002`
- primary asset: one promotional hero that makes the offer legible
- secondary asset: one tighter promotional variant for a smaller surface
- supporting asset: one message-consistent companion asset that preserves the same communication purpose without repeating the hero exactly

The point of this set is not campaign storytelling and not grouped merchandising context. The point is to test whether one targeted promotional message can remain legible, bounded, and surface-usable across a small asset family while still keeping product presence credible.

This keeps the proof closer to message-bearing communication discipline than to campaign-world coherence or collection grouping logic.

## Distinct Burden Of This Mode

The distinct burden for this proof is:

- the promotional offer or message-bearing signal must persist across the full small asset set
- the set must still read as one controlled communication unit rather than as three loosely related branded images
- bounded promotional variation must remain allowed without weakening the targeted communication purpose
- product presence must stay credible enough that the message set does not collapse into obvious distortion or empty promotional overlay
- surface usability must hold across the intended small set without drifting into either campaign looseness or single-SKU documentation

The key tension to test is:

- targeted communication purpose pulling against bounded asset-level variation

That is the distinct burden this proof should carry. The live question is whether the current Airtable proof shape can hold one small promotional family whose message-bearing purpose survives cleanly across multiple assets without requiring pre-adjustment.

## Role Of Review Context

This proof should treat review context as useful and more deliberate than the simplest SKU-side case, but not campaign-style load-bearing.

The planned review context is:

- one side-by-side message-check review board comparing the three promotional assets
- one light annotation layer only where message placement, emphasis, or offer clarity need to be called out for review clarity

That review context should help make message persistence and promotional discipline legible across the set.

It should not become the primary readiness burden in the stronger campaign / editorial sense. The governed outputs still need to stand on their own as a usable promotional communication set.

This keeps the proof honest to current repo-local evidence, which supports message-check comparison as useful here without yet supporting campaign-style review-board doctrine for this mode.

## How This Burden Differs Materially From SKU, Campaign, And Collection

This proof should frame its burden concretely rather than abstractly.

Relative to SKU-driven product imagery:

- the governing burden is not single-item fidelity and repeatable comparability alone
- the governing burden is whether the targeted promotion survives across a small asset family without product presence becoming untrustworthy

Relative to campaign / editorial:

- the governing burden is not broader campaign-family coherence, directional world-building, or expressive continuity across a mini-family
- the governing burden is tighter and more instrumental: the same offer and message-bearing signals must remain legible across controlled promotional surfaces

Relative to collection / merchandising:

- the governing burden is not grouped-set adjacency, spacing, or relationship-bearing truth across multiple items
- the governing burden is message persistence across a promotional set whose members serve one communication purpose rather than one grouped arrangement

The bounded scenario should therefore test:

- governed message persistence, not single-SKU comparability
- governed communication discipline, not campaign-family coherence
- governed promotional continuity, not grouped-set relational legibility

Planning-level answer: yes, this bounded marketing / message-driven proof should be attempted on the current packet logic and current proof shape without pre-adjustment.

That answer is narrow and provisional.

It does not mean the current packet logic is already proven sufficient for marketing / message-driven work in general.

It means only that current repo-local truth still supports one honest attempt inside the existing Airtable surface because:

- the repo still appears to share one packet-and-seam model across distinct burden shapes
- marketing / message-driven already has a concrete packet example, seam-bridge framing, and one negative output-boundary note in repo-local docs
- the proposed proof remains bounded to one message unit, one anchor product, one small three-asset set, one first-pass seam trail, and useful-but-not-load-bearing review context

The live proof question is therefore not whether marketing / message-driven deserves schema pre-adjustment in theory.

The live proof question is whether one bounded promotional communication packet can be declared and carried honestly enough on the current surface to yield interpretable first-pass findings.
